আজ আমরা Laravel এ Routing সম্পর্কে শিখব। Routing হলো একটি প্রক্রিয়া যা url এবং controller method এর মধ্যে সংযোগ স্থাপন করে। ইউজার যদি কোনো একটা রাউটে ক্লিক করে, তাহলে সেই রাউটের সাথে সংযুক্ত controller method টি কল করা হয়। আমরা রাউট কেন ব্যবহার করি? কারণ এটি আমাদের অ্যাপ্লিকেশনের বিভিন্ন অংশে ইউজারদের নেভিগেট করতে সাহায্য করে এবং বিভিন্ন ফাংশনালিটি প্রদান করে। এর সাথে আমরা middleware দিয়ে রাউট গুলোকে ফিল্টার করতে পারি। laravel 12 কিন্তু api এর জন্য রাউট ফাইল দেয়া থাকে না, আপনাকে নিজে তৈরি করতে হবে। 

আমরা url এর মাধ্যমেও রাউট করতে পারি। কিন্তু এখানে কিছু বাধ্যবাধকতা আছে। যেমন ধরা যাক, আপনার একটি রাউট আছে, সেটা অনেক জায়গায় কল হতে পারে। তখন যদি আপনি সেই রাউটের url পরিবর্তন করেন, তাহলে আপনাকে সব জায়গায় গিয়ে সেই url পরিবর্তন করতে হবে। কিন্তু যদি আপনি রাউটের নাম দিয়ে রাউট করেন, তাহলে শুধু রাউট ফাইলে গিয়ে রাউটের নাম এর সাথে নতুন url টি সংযুক্ত করে দিলেই হবে। তো, এটি হচ্ছে রাউট এবং নেম রাউট। 

আমাদের http request এর জন্য বেশ কিছু মেথড আছে, যেমন GET, POST, PUT, DELETE ইত্যাদি। আমরা এই মেথড গুলো ব্যবহার করে রাউট তৈরি করতে পারি। এগুলো কোনটা কি কাজ করে তা নিচে দেওয়া হলোঃ
1. GET: এই মেথডটি ডাটা রিট্রিভ করার জন্য ব্যবহার করা হয়।
2. POST: এই মেথডটি নতুন ডাটা তৈরি করার জন্য ব্যবহার করা হয়।
3. PUT: এই মেথডটি বিদ্যমান ডাটা আপডেট করার জন্য ব্যবহার করা হয়।
4. DELETE: এই মেথডটি ডাটা মুছে ফেলার জন্য ব্যবহার করা হয়। 
5. PATCH: এই মেথডটি ডাটার আংশিক আপডেট করার জন্য ব্যবহার করা হয়। 

চলুন এতক্ষণ যা শিখলাম তা উদাহরণ দিয়ে বুঝি। ধরুন আমাদের একটি ব্লগ সাইট আছে, যেখানে ইউজাররা বিভিন্ন পোস্ট দেখতে পারে, নতুন পোস্ট তৈরি করতে পারে, বিদ্যমান পোস্ট আপডেট করতে পারে এবং পোস্ট মুছে ফেলতে পারে। আমরা এই কাজগুলো করার জন্য নিচের রাউট গুলো তৈরি করবঃ

```php
use App\Http\Controllers\PostController;
use Illuminate\Support\Facades\Route;

// সকল পোস্ট দেখার রাউট
Route::get('/posts', [PostController::class, 'index'])->name('posts.index');
// নতুন পোস্ট তৈরি করার রাউট
Route::post('/posts', [PostController::class, 'store'])->name('posts.store');
// একটি নির্দিষ্ট পোস্ট দেখার রাউট
Route::get('/posts/{id}', [PostController::class, 'show'])->name('posts.show');
// একটি নির্দিষ্ট পোস্ট আপডেট করার রাউট
Route::put('/posts/{id}', [PostController::class, 'update'])->name('posts.update');
// একটি নির্দিষ্ট পোস্ট মুছে ফেলার রাউট
Route::delete('/posts/{id}', [PostController::class, 'destroy'])->name('posts.destroy');
```
এখানে আমরা একটি PostController ব্যবহার করেছি, যা আমাদের ব্লগ পোস্ট গুলো পরিচালনা করবে। প্রতিটি রাউটের সাথে একটি নামও সংযুক্ত করা হয়েছে, যা আমাদের রাউট গুলোকে সহজে রেফারেন্স করতে সাহায্য করবে। 

লারাভেল এর নতুন ভার্সনে আমরা api রাউট ব্যবহার করতে হলে কমান্ডের মাধ্যমে ফাইলটি বানিয়ে নিতে হবে। নিচের কমান্ডটি ব্যবহার করুনঃ

```bash
php artisan install:api 
```
এই কমান্ডটি চালানোর পর আপনার প্রজেক্টের routes ফোল্ডারে একটি api.php ফাইল তৈরি হবে, যেখানে আপনি আপনার API রাউট গুলো সংজ্ঞায়িত করতে পারবেন। 
এবার আমরা দেখবো কিভাবে একটি কাস্টম রাউট ফাইল তৈরি করতে হয়। ধরুন আমরা একটি নতুন রাউট ফাইল তৈরি করতে চাই যার নাম হবে admin.php। আমরা নিচের ধাপগুলো অনুসরণ করবঃ 
1. প্রথমে routes ফোল্ডারে একটি নতুন ফাইল তৈরি করুন যার নাম হবে admin.php।
2. এরপর এই ফাইলে আপনার রাউট গুলো সংজ্ঞায়িত করুন, যেমনঃ

```php
use App\Http\Controllers\AdminController;
use Illuminate\Support\Facades\Route;
Route::get('/dashboard', [AdminController::class, 'dashboard'])->name('admin.dashboard');
```
3. এরপর আপনাকে এই নতুন রাউট ফাইলটি লোড করতে হবে। এজন্য আপনাকে আপনার bootstrap\app.php ফাইলে গিয়ে নিচের কোডটি যোগ করতে হবেঃ

```php
use App\Http\Middleware\AdultMiddleware;
use Illuminate\Foundation\Application;
use Illuminate\Foundation\Configuration\Exceptions;
use Illuminate\Foundation\Configuration\Middleware;

return Application::configure(basePath: dirname(__DIR__))
    ->withRouting(
        web: [
            __DIR__ . '/../routes/web.php',
            __DIR__ . '/../routes/admin.php',
        ],
        commands: __DIR__ . '/../routes/console.php',
        health: '/up',
    )
    ->withMiddleware(function (Middleware $middleware): void {
        $middleware->alias([
            'adult' => AdultMiddleware::class,
        ]);
    })
    ->withExceptions(function (Exceptions $exceptions): void {
        //
    })->create();
```
এখন আপনার admin.php ফাইলে সংজ্ঞায়িত রাউট গুলো কাজ করবে। 
এখন চলুন, আমরা Route Parameters সম্পর্কে শিখি। Route Parameters হলো এমন একটি প্রক্রিয়া যার মাধ্যমে আমরা রাউটের মধ্যে ডায়নামিক ভ্যালু পাস করতে পারি। ধরুন আমাদের একটি রাউট আছে যা একটি নির্দিষ্ট পোস্ট দেখাবে, তখন আমরা সেই পোস্টের আইডি রাউটের মধ্যে পাস করতে পারি। নিচে একটি উদাহরণ দেওয়া হলোঃ

```php
Route::get('/posts/{id}', [PostController::class, 'show'])->name('posts.show');
```
এখানে {id} হলো একটি রাউট প্যারামিটার, যা ডায়নামিক ভ্যালু গ্রহণ করবে। যখন ইউজার এই রাউটে যাবে, তখন সেই আইডি ভ্যালুটি PostController এর show মেথডে পাস করা হবে।

## মূল ধারণা ও প্রয়োজনীয় ইনসাইট

### **রাউটিং: সংজ্ঞা ও উদ্দেশ্য**

Laravel-এ রাউটিং হলো এমন একটি প্রক্রিয়া, যা নির্দিষ্ট URL-কে সংশ্লিষ্ট লজিক বা কন্ট্রোলারের সাথে যুক্ত করে। যখন কোনো ব্যবহারকারী `/about` অথবা `/contact` এর মতো URL ভিজিট করে, তখন Laravel সেই অনুরোধকে সঠিক কন্ট্রোলার বা কলব্যাক ফাংশনে পাঠায় এবং প্রয়োজনীয় কনটেন্ট রেন্ডার করে।

* রাউটকে বলা যায় URL ও কন্ট্রোলার লজিকের মধ্যকার “সেতুবন্ধন”।
* SEO–ফ্রেন্ডলি URL গঠন করতে রাউটিং গুরুত্বপূর্ণ ভূমিকা রাখে—বিশেষত ই-কমার্স, নিউজ পোর্টাল বা বড় ওয়েবসাইটগুলোর ক্ষেত্রে।
* রাউটে Middleware প্রয়োগ করা যায়, যা কন্ট্রোলারে পৌঁছানোর আগে বা পরে অনুরোধকে ফিল্টার করতে সক্ষম।

### **রাউট ব্যবহারের মৌলিক নিয়ম**

* Laravel–এ রাউট সাধারণত `web.php`, `api.php` অথবা কাস্টম রাউট ফাইলের মধ্যে ডিক্লেয়ার করা হয়।
* উদাহরণ: `/about` URL অ্যাক্সেস করলে একটি নির্দিষ্ট ভিউ রিটার্ন করা।
* URL জেনারেট করতে `url()` ফাংশন এবং আরও রক্ষণযোগ্য কোড লেখার জন্য `route()` ফাংশন ব্যবহার করা হয়—বিশেষত Named Route ব্যবস্থায়।

### **Named Route বনাম URL ফাংশন**

* `url()` সরাসরি URL স্ট্রিং অনুসরণ করে—URL পরিবর্তন হলে একাধিক জায়গায় আপডেট দরকার হয়।
* `route()` নির্ভর করে রাউটের নামের ওপর—ফলে URL পরিবর্তন হলেও কোডবেস ক্ষতিগ্রস্ত হয় না।
* Named Route কোডকে আরও পাঠযোগ্য ও রক্ষণযোগ্য করে।

## Laravel Routing-এ বহুল ব্যবহৃত HTTP Methods

ভিডিওতে Laravel রাউটিং-এ ব্যবহৃত সবচেয়ে প্রচলিত HTTP মেথডগুলো ব্যাখ্যা করা হয়েছে:

| Method     | উদ্দেশ্য            | উদাহরণ                             |
| ---------- | ------------------- | ---------------------------------- |
| **GET**    | ডাটা রিট্রিভ        | কোনো ওয়েবপেজ দেখানো বা ডাটা ফেচ    |
| **POST**   | নতুন ডাটা তৈরি      | ফর্ম সাবমিট করে রেকর্ড তৈরি        |
| **PUT**    | সম্পূর্ণ ডাটা আপডেট | একটি রিসোর্স পুরোপুরি পরিবর্তন     |
| **PATCH**  | আংশিক আপডেট         | নির্দিষ্ট ফিল্ড বা আংশিক পরিবর্তন  |
| **DELETE** | ডাটা ডিলিট          | ডাটাবেস থেকে কোনো রেকর্ড মুছে ফেলা |

## কাস্টম রাউট ফাইল

Laravel-এ প্রয়োজন অনুযায়ী `admin.php`–এর মতো কাস্টম রাউট ফাইল তৈরি করা যায়।

* এগুলো সাধারণত ফিচার, মডিউল বা ইউজার রোল অনুযায়ী রাউট সংগঠিত করতে ব্যবহৃত হয়।
* এই ধরনের রাউট ফাইলগুলোকে লারাভেলের বুটস্ট্র্যাপিং প্রসেসে Explicitভাবে নিবন্ধন করতে হয়।
* বড় স্কেলের অ্যাপ্লিকেশনে রাউট ব্যবস্থাপনা সহজ করতে modular structure অত্যন্ত কার্যকর।

## Laravel 12-এ API Routing

Laravel 12–এ আগের মতো ডিফল্ট `api.php` ফাইল থাকে না।

* API ডেভেলপ করতে চাইলে ডেভেলপারকে নিজে থেকে API রাউট তৈরি বা ইনস্টল করতে হবে।
* ভিডিওতে উল্লেখ রয়েছে যে ভবিষ্যতের টিউটোরিয়ালে API Routing ও Token-Based Authentication বিস্তারিত আলোচনা হবে।

## রাউট প্যারামিটার ও ডাইনামিক রাউটিং

* Laravel রাউটে বিভিন্ন প্যারামিটার গ্রহণ করা যায়—যেমন ID—যা কন্ট্রোলারে ডায়নামিক ডাটা পাঠাতে ব্যবহৃত হয়।
* উদাহরণ: `/students/5` অ্যাক্সেস করলে ID = 5 ছাত্রের প্রোফাইল দেখানো।
* RESTful API ও Resourceful Routing কাঠামো বুঝতে এই ধারণা অপরিহার্য।
