### ১. সার্ভার আপডেট করা

```bash
sudo apt update && sudo apt upgrade -y

```

*(পাসওয়ার্ড চাইলে আপনার VPS এর পাসওয়ার্ডটি দিন। মনে রাখবেন, টাইপ করার সময় পাসওয়ার্ড দেখা যাবে না। আপডেট শেষ হতে ১-২ মিনিট সময় নিতে পারে।)*

যেহেতু আপনার স্ক্রিনে **"System restart required"** লেখা ছিল, তাই আপডেট শেষ হলে একবার রিবুট দেওয়া ভালো:

```bash
sudo reboot

```

*(রিবুট দিলে আপনি সার্ভার থেকে ডিসকানেক্ট হয়ে যাবেন। ১ মিনিট পর আবার আগের মতো SSH দিয়ে লগইন করুন।)*

### ২. লারাভেলের জন্য প্রয়োজনীয় সফটওয়্যার ইন্সটল (LEMP Stack)

লারাভেল চালানোর জন্য আমাদের PHP, Nginx এবং MySQL লাগবে। নিচের বড় কমান্ডটি কপি করে রান করুন, এটি সব গুরুত্বপূর্ণ এক্সটেনশনসহ PHP ইন্সটল করে দেবে:

```bash
sudo apt install nginx mysql-server php8.3-fpm php8.3-mysql php8.3-common php8.3-xml php8.3-bcmath php8.3-curl php8.3-mbstring php8.3-zip php8.3-intl php8.3-gd unzip curl git -y

```

### ৩. কম্পোজার (Composer) ইন্সটল করা

```bash
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

```

ইন্সটল হয়েছে কি না চেক করতে লিখুন: `composer -v` (কম্পোজারের লোগো আসলে বুঝবেন কাজ হয়েছে)।

### ৪. ডাটাবেস তৈরি করা

এখন আপনার ইনভেন্টরি প্রজেক্টের জন্য একটি ডাটাবেস তৈরি করতে হবে।

* প্রথমে MySQL এ ঢুকুন: `sudo mysql`
* তারপর নিচের কমান্ডগুলো একে একে লিখুন (প্রতি লাইনের শেষে সেমিকোলন `;` দিতে ভুলবেন না):

```sql
CREATE DATABASE inventory_db;
CREATE USER 'inventory_user'@'localhost' IDENTIFIED BY 'your_password_here';
GRANT ALL PRIVILEGES ON inventory_db.* TO 'inventory_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

```

*(আপনার পছন্দমতো ডাটাবেস নাম এবং পাসওয়ার্ড দিন।)*

আপনি কি আপনার প্রজেক্টটি গিটহাব থেকে `clone` করতে পেরেছেন? যদি প্রজেক্টটি **Private** হয়, তাহলে গিটহাব আপনার কাছে ইউজারনেম এবং **Personal Access Token** (পাসওয়ার্ডের বদলে) চাইতে পারে।

আপনি এই ধাপগুলো শেষ করলে আমাকে জানান, আমি আপনাকে শিখিয়ে দেব কীভাবে **Nginx** কনফিগার করতে হয় যাতে আপনার আইপি (`202.181.12.25`) ব্রাউজারে লিখলে প্রজেক্টটি দেখা যায়। 










অভিনন্দন! আপনি সফলভাবে আপনার VPS-এ লগইন করেছেন। আপনি এখন উবুন্টু (Ubuntu 24.04) সার্ভারের টার্মিনালে আছেন। আপনার ইউজারনেম হচ্ছে `arkpowerweb` এবং এটি একটি একদম ফ্রেশ সার্ভার।

শুরুতেই আমরা সার্ভারটিকে প্রস্তুত করে নেব। নিচের কমান্ডগুলো একটি একটি করে কপি করে টার্মিনালে পেস্ট করুন এবং এন্টার দিন:

### ১. সার্ভার আপডেট করা

প্রথমে সিস্টেমের প্যাকেজ লিস্ট আপডেট করে নিন:

```bash
sudo apt update && sudo apt upgrade -y

```

*(পাসওয়ার্ড চাইলে আপনার VPS এর পাসওয়ার্ডটি দিন। মনে রাখবেন, টাইপ করার সময় পাসওয়ার্ড দেখা যাবে না। আপডেট শেষ হতে ১-২ মিনিট সময় নিতে পারে।)*

যেহেতু আপনার স্ক্রিনে **"System restart required"** লেখা ছিল, তাই আপডেট শেষ হলে একবার রিবুট দেওয়া ভালো:

```bash
sudo reboot

```

*(রিবুট দিলে আপনি সার্ভার থেকে ডিসকানেক্ট হয়ে যাবেন। ১ মিনিট পর আবার আগের মতো SSH দিয়ে লগইন করুন।)*

---

### ২. লারাভেলের জন্য প্রয়োজনীয় সফটওয়্যার ইন্সটল (LEMP Stack)

লারাভেল চালানোর জন্য আমাদের PHP, Nginx এবং MySQL লাগবে। নিচের বড় কমান্ডটি কপি করে রান করুন, এটি সব গুরুত্বপূর্ণ এক্সটেনশনসহ PHP ইন্সটল করে দেবে:

```bash
sudo apt install nginx mysql-server php8.3-fpm php8.3-mysql php8.3-common php8.3-xml php8.3-bcmath php8.3-curl php8.3-mbstring php8.3-zip php8.3-intl php8.3-gd unzip curl git -y

```

---

### ৩. কম্পোজার (Composer) ইন্সটল করা

লারাভেলের প্যাকেজ ম্যানেজ করার জন্য কম্পোজার দরকার:

```bash
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

```

ইন্সটল হয়েছে কি না চেক করতে লিখুন: `composer -v` (কম্পোজারের লোগো আসলে বুঝবেন কাজ হয়েছে)।

---

### ৪. ডাটাবেস তৈরি করা

এখন আপনার ইনভেন্টরি প্রজেক্টের জন্য একটি ডাটাবেস তৈরি করতে হবে।

* প্রথমে MySQL এ ঢুকুন: `sudo mysql`
* তারপর নিচের কমান্ডগুলো একে একে লিখুন (প্রতি লাইনের শেষে সেমিকোলন `;` দিতে ভুলবেন না):

```sql
CREATE DATABASE inventory_db;
CREATE USER 'inventory_user'@'localhost' IDENTIFIED BY 'your_password_here';
GRANT ALL PRIVILEGES ON inventory_db.* TO 'inventory_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

```

*(আপনার পছন্দমতো ডাটাবেস নাম এবং পাসওয়ার্ড দিন।)*

### ৫. গিটহাব থেকে প্রজেক্ট আনা

এখন আমরা প্রজেক্টটি ডাউনলোড করবো:

```bash
cd /var/www
sudo chown arkpowerweb:arkpowerweb /var/www
git clone https://github.com/your-username/your-repo.git inventory
cd inventory

```

**আপনার জন্য পরবর্তী প্রশ্ন:**
আপনি কি আপনার প্রজেক্টটি গিটহাব থেকে `clone` করতে পেরেছেন? যদি প্রজেক্টটি **Private** হয়, তাহলে গিটহাব আপনার কাছে ইউজারনেম এবং **Personal Access Token** (পাসওয়ার্ডের বদলে) চাইতে পারে।

আপনি এই ধাপগুলো শেষ করলে আমাকে জানান, আমি আপনাকে শিখিয়ে দেব কীভাবে **Nginx** কনফিগার করতে হয় যাতে আপনার আইপি (`202.181.12.25`) ব্রাউজারে লিখলে প্রজেক্টটি দেখা যায়। 









অবশ্যই! লিনাক্স (Linux) সার্ভারে ঘোরাফেরা করার জন্য আপনাকে মাউস নয়, কিছু ছোট ছোট কমান্ড ব্যবহার করতে হবে। এগুলো জানলে আপনি খুব সহজে ফোল্ডার চেক করতে পারবেন।

নিচে আপনার জন্য সবচেয়ে প্রয়োজনীয় কমান্ডগুলো দেওয়া হলো:

---

### ১. ফোল্ডারের ভেতর কী আছে তা দেখা (`ls`)

আপনি এখন যে ফোল্ডারে আছেন, তার ভেতরে কী কী ফাইল বা ফোল্ডার আছে তা দেখতে টাইপ করুন:

* `ls` (শুধু নামগুলো দেখাবে)
* `ls -la` (ফাইল সাইজ, পারমিশন এবং লুকানো ফাইলসহ সব দেখাবে। এটিই সবচেয়ে বেশি ব্যবহার করবেন)।

### ২. লোকেশন চেক করা (`pwd`)

অনেক সময় কাজ করতে করতে ভুলে যেতে পারেন আপনি এখন কোথায় আছেন। তখন টাইপ করুন:

* `pwd` (Print Working Directory) - এটি আপনাকে পূর্ণ পাথ (যেমন: `/var/www/inventory`) দেখিয়ে দেবে।

### ৩. এক ফোল্ডার থেকে অন্য ফোল্ডারে যাওয়া (`cd`)

* `cd /var/www` (সরাসরি ওই ফোল্ডারে চলে যাবে)
* `cd ..` (এক ধাপ পেছনে বা উপরের ফোল্ডারে যাবে)
* `cd ~` (আপনার ইউজারের হোম ডিরেক্টরিতে ফিরে আসবে)

---

### লিনাক্স সার্ভারের প্রধান ফোল্ডারগুলোর পরিচিতি

আপনি যখন `cd /` দিয়ে একদম শুরুতে (Root) যাবেন, তখন নিচের ফোল্ডারগুলো দেখতে পাবেন। লারাভেল ডেভেলপার হিসেবে আপনার যা যা চেনা দরকার:

| ফোল্ডার                    | কেন গুরুত্বপূর্ণ?                                        |
| ----------------------- | -------------------------------------------------- |
| **`/var/www`**          | এখানে সাধারণত আপনার লারাভেল প্রজেক্টের কোড থাকবে।                 |
| **`/etc/nginx`**        | এখানে Nginx এর কনফিগারেশন ফাইল থাকে (ডোমেইন সেটআপের জন্য)।      |
| **`/etc/php`**          | PHP এর সেটিংস বা `php.ini` ফাইল এখানে থাকে।                 |
| **`/home/arkpowerweb`** | এটি আপনার ব্যক্তিগত ফোল্ডার। এখানে আপনি ফাইল ডাউনলোড করে রাখতে পারেন। |
| **`/var/log`**          | সার্ভারে কোনো সমস্যা হলে তার রিপোর্ট বা লগ এখানে জমা হয়।            |

---

### ৪. ফাইল পড়া এবং এডিট করা (`nano` বা `cat`)

ফাইল না খুলে শুধু ভেতরে কী আছে তা দেখতে:

* `cat .env` (পুরো ফাইল টার্মিনালে প্রিন্ট করবে)

আর যদি ফাইলের ভেতরে কিছু লিখতে বা পরিবর্তন করতে চান (যেমন `.env` এ ডাটাবেস পাসওয়ার্ড বসানো):

* `nano .env` (এটি একটি টেক্সট এডিটর ওপেন করবে। এডিট শেষ হলে `Ctrl+O` দিয়ে সেভ এবং `Ctrl+X` দিয়ে বের হতে হয়)।

---

### ৫. ফাইল বা ফোল্ডার তৈরি ও ডিলিট করা

* `mkdir test_folder` (নতুন ফোল্ডার তৈরি)
* `rm filename.txt` (ফাইল ডিলিট - **সাবধান!** এটি ডিলিট করলে আর ফিরে পাওয়া যায় না)

---

**আপনার জন্য একটি ছোট প্র্যাকটিস:**
১. প্রথমে `cd /var/www` লিখে এন্টার দিন।
২. সেখানে `ls -la` দিয়ে দেখুন কোনো ফোল্ডার আছে কি না।
৩. তারপর `cd ~` লিখে আপনার হোম ডিরেক্টরিতে ফিরে আসুন।

এই কমান্ডগুলো কি বুঝতে পেরেছেন? ট্রাই করে দেখুন, কোনো এরর আসলে আমাকে জানান। এরপর আমরা প্রজেক্ট ক্লোন (Clone) করার কাজ শুরু করবো। 









চমৎকার! আপনি একদম সার্ভারের মেইন ডিরেক্টরিতে (Root `/`) আছেন। কিন্তু আপনি যখন `mkdir learning_vps` দিয়েছেন, তখন সার্ভার আপনাকে **"Permission denied"** বলেছে।

কেন এটা হলো? এটা আপনার শেখার জন্য খুব গুরুত্বপূর্ণ একটা পাঠ।

### লিনাক্স পারমিশন ও 'sudo' এর ব্যবহার

আপনার লিনাক্স সার্ভারে আপনি এখন `arkpowerweb` নামক ইউজার হিসেবে আছেন। এই ইউজারের নিজের বাড়ি (Home) হলো `/home/arkpowerweb` ফোল্ডার। সেখানে সে যা খুশি করতে পারে। কিন্তু সার্ভারের মেইন ফাইল সিস্টেম বা অন্যান্য সিস্টেমে কাজ করতে হলে আপনার **"Administrative Power"** বা ক্ষমতা লাগে।

লিনাক্সে এই ক্ষমতাকে বলা হয় **sudo** (SuperUser DO)।

**সমাধান:**
আপনি যদি মেইন ডিরেক্টরিতে ফোল্ডার বানাতে চান, তবে কমান্ডের আগে `sudo` বসাতে হবে:

```bash
sudo mkdir learning_vps

```

তবে আমি আপনাকে পরামর্শ দেব, মেইন ডিরেক্টরিতে এলোমেলো ফোল্ডার না বানিয়ে আপনার নিজের হোমে ফিরে যেতে। সেখানে কোনো `sudo` লাগবে না।

১. **নিজের হোমে ফিরুন:** ```bash
cd ~

```
(এখন দেখুন আপনার নীল রঙের লেখাটি `arkpowerweb@arkpowerweb:~$` হয়ে গেছে। ওই `~` মানেই আপনি এখন আপনার এলাকায় আছেন।)

২. **এখন কমান্ডটি আবার ট্রাই করুন:**
```bash
mkdir learning_vps
ls

```

(এবার আর কোনো এরর দেবে না!)

---

### এবার আমাদের আসল প্রজেক্টের জায়গায় যাই

লারাভেল বা যেকোনো ওয়েব প্রজেক্ট রাখার স্ট্যান্ডার্ড জায়গা হলো `/var/www` ফোল্ডার। চলুন সেখানে যাই:

```bash
cd /var/www
ls

```

সেখানে সম্ভবত আপনি `html` নামে একটি ফোল্ডার দেখতে পাবেন। এটি Nginx এর ডিফল্ট ফোল্ডার।

---

**আপনার জন্য পরের কাজ:**
আমরা এই `/var/www` ফোল্ডারের ভেতর আমাদের প্রজেক্টটি গিটহাব থেকে ক্লোন করবো। কিন্তু তার আগে আমাদের এই ফোল্ডারটির মালিকানা (Ownership) আপনার ইউজারকে দিয়ে দিতে হবে, যাতে বারবার `sudo` টাইপ করতে না হয়।

নিচের কমান্ডটি দিন (এটি `/var/www` এর মালিকানা আপনার ইউজারকে দেবে):

```bash
sudo chown -R arkpowerweb:arkpowerweb /var/www

```

এটি করার পর একবার `ls -la` দিয়ে দেখুন তো `/var/www` ফোল্ডারের পাশে `root` লেখা নাকি আপনার ইউজারনেম `arkpowerweb` লেখা?

মালিকানা ঠিক হয়ে গেলে আমরা **Git Clone** করার জন্য প্রস্তুত! আপনার কি গিটহাব রিপোজিটরি লিঙ্কটি রেডি আছে? 










চমৎকার! আপনি খুব দ্রুত বিষয়গুলো আয়ত্ত করে ফেলছেন। **sudo** এর মানে হচ্ছে আপনি সার্ভারের "বস" হিসেবে কমান্ড দিচ্ছেন।

যেহেতু প্রজেক্টটি **Private**, তাই আমাদের একটি **Personal Access Token (PAT)** লাগবে। চলুন ধাপে ধাপে করি:

### ধাপ ১: গিটহাব থেকে টোকেন (PAT) সংগ্রহ করা

১. আপনার গিটহাব একাউন্টের **Settings**-এ যান।
২. বাম দিকের একদম নিচে **Developer settings**-এ ক্লিক করুন।
৩. **Personal access tokens** > **Tokens (classic)**-এ যান।
৪. **Generate new token (classic)**-এ ক্লিক করুন।
৫. 'Note'-এ লিখুন "My VPS Server"। 'Expiration' দিন ৯০ দিন বা আপনার ইচ্ছা মতো।
৬. **Select scopes:** শুধু `repo` বক্সে টিক চিহ্ন দিন।
৭. একদম নিচে **Generate token**-এ ক্লিক করুন।
৮. **সাবধান:** টোকেনটি কপি করে কোথাও সেভ করে রাখুন। এটি একবারই দেখাবে, হারিয়ে ফেললে আবার নতুন করে বানাতে হবে।

এখন সার্ভারে ফিরে যান। আমরা শুধু `production` ব্রাঞ্চটি ক্লোন করবো। নিচের কমান্ডটি সাজিয়ে লিখুন:

```bash
cd /var/www
git clone -b production https://github.com/your-username/your-repo-name.git inventory

```

*(এখানে `your-username/your-repo-name` পরিবর্তন করে আপনার প্রজেক্টের পাথ দিন।)*

**যখন ইউজারনেম ও পাসওয়ার্ড চাইবে:**

* **Username:** আপনার গিটহাব ইউজারনেম দিন।
* **Password:** গিটহাব থেকে কপি করা সেই **Token** টি পেস্ট করুন। (পেস্ট করার সময় স্ক্রিনে কিছু দেখা যাবে না, এটা নিয়ে ঘাবড়াবেন না, পেস্ট করে এন্টার দিন)।

**আপনার পরবর্তী কাজ:**
১. টোকেন জেনারেট করে প্রজেক্টটি ক্লোন করুন।

### ৫. একটি প্রো-টিপ: সার্ভিসগুলো সচল আছে কি না দেখা

যদি দেখেন কোনোটি ইন্সটল করা আছে, তবে সেটি বর্তমানে চালু (Running) আছে কি না তা দেখতে এই কমান্ডটি দিন:

```

এখানে একটি মজার বিষয় দেখুন: আপনার সার্ভারে **Apache** এবং **Nginx** দুটিই ইন্সটল করা আছে। সাধারণত আমরা যেকোনো একটি ব্যবহার করি। তবে যেহেতু PHP 8.4 এবং Composer রেডি আছে, আপনার কাজ অনেক সহজ হয়ে গেল।

এখন আমাদের **Mission** হলো কোড নামানো এবং কনফিগার করা। যেহেতু আপনি `sudo` ছাড়া কাজ করার পারমিশন আগেই সেট করেছেন, চলুন ধাপে ধাপে আগাই।

### ধাপ ১: প্রজেক্ট ক্লোন করা (Production Branch)

প্রথমে `/var/www` ফোল্ডারে গিয়ে আপনার সেই প্রাইভেট রিপোজিটরি ক্লোন করুন।

*(ইউজারনেম চাইলে আপনার গিটহাব ইউজারনেম দেবেন, আর পাসওয়ার্ড চাইলে আপনার তৈরি করা **Personal Access Token** টি পেস্ট করবেন।)*

### ধাপ ২: ডিপেন্ডেন্সি ইন্সটল করা (Composer)

প্রজেক্ট ফোল্ডারে ঢুকে লারাভেলের প্রয়োজনীয় প্যাকেজগুলো ইন্সটল করতে হবে:

*(এটি করতে ২-৩ মিনিট সময় নিতে পারে।)*

### ধাপ ৩: এনভায়রনমেন্ট ফাইল সেটআপ

লারাভেলের জন্য `.env` ফাইল খুবই গুরুত্বপূর্ণ। আমরা `.env.example` থেকে একটি কপি বানিয়ে নেব:

```bash
cp .env.example .env

```

এখন ফাইলটি এডিট করে ডাটাবেস সেটআপ করতে হবে:

```bash
nano .env

```

এখানে নিচের লাইনগুলো খুঁজে বের করে আপনার ডাটাবেসের তথ্য দিন (আমি আগের মেসেজে আপনাকে ডাটাবেস বানাতে শিখিয়েছিলাম):

* `DB_DATABASE=inventory_db`
* `DB_USERNAME=inventory_user`
* `DB_PASSWORD=your_password`

*(সেভ করতে `Ctrl+O`, তারপর `Enter`, বের হতে `Ctrl+X`)*

### ধাপ ৪: অ্যাপ কি (App Key) এবং পারমিশন

লারাভেলের সিকিউরিটির জন্য কি জেনারেট করতে হয়:

php artisan key:generate

sudo chmod -R 775 storage bootstrap/cache
sudo chown -R www-data:www-data storage bootstrap/cache

```

### ধাপ ৫: ডাটাবেস মাইগ্রেশন

আপনার টেবিলগুলো ডাটাবেসে পাঠানোর জন্য এই কমান্ডটি দিন:

```bash
php artisan migrate

```

---

**এখন একটি গুরুত্বপূর্ণ প্রশ্ন:**
আপনার সার্ভারে Apache এবং Nginx দুটোই আছে। ডিফল্টভাবে বর্তমানে কোনটি পোর্ট ৮০ (Port 80) দখল করে আছে তা জানতে ব্রাউজারে আপনার আইপি (`202.181.12.25`) লিখে এন্টার দিন।

* যদি **Apache2 Ubuntu Default Page** আসে, তবে আমরা **Apache** কনফিগার করবো।
* যদি **Welcome to nginx!** আসে, তবে আমরা **Nginx** কনফিগার করবো।

ব্রাউজারে কী দেখতে পাচ্ছেন? সেটি জানালে আমি আপনাকে **Virtual Host** সেটআপ করা শিখিয়ে দেব, যাতে আইপি দিলেই আপনার লারাভেল প্রজেক্টটি লোড হয়। 










হ্যাঁ, দুটো একসাথে থাকলে **প্রবলেম হয়**। বিশেষ করে যখন আপনি আইপি দিয়ে ব্রাউজারে সাইটটি দেখতে যাবেন, তখন তারা একে অপরের সাথে মারামারি শুরু করবে।

### কেন প্রবলেম হয়?

যেকোনো ওয়েব সার্ভার (Apache বা Nginx) ডিফল্টভাবে **পোর্ট ৮০ ()** ব্যবহার করে। একটি পোর্টে একবারে একটি মাত্র সফটওয়্যার চলতে পারে। ফলে:
১. যে আগে স্টার্ট হবে সে পোর্ট দখল করে নেবে।
২. অন্যটি স্টার্ট হতে গিয়ে "Address already in use" এরর দেখাবে।

---

### আপনার জন্য পরামর্শ:

লারাভেলের জন্য বর্তমান সময়ে **Nginx** ব্যবহার করা সবচেয়ে বুদ্ধিমানের কাজ। এটি আধুনিক, ফাস্ট এবং আপনার সিনিয়র সম্ভবত আগের সার্ভারে এটিই ব্যবহার করেছিলেন।

তাই আমরা এখন **Apache-কে বন্ধ করে দেব** এবং **Nginx-কে মূল সার্ভার হিসেবে ব্যবহার করব**।

#### ধাপ ১: Apache বন্ধ করা ও ডিজাবেল করা

নিচের কমান্ডগুলো একটি একটি করে দিন:

```bash
sudo systemctl stop apache2
sudo systemctl disable apache2

```

*(এটি করলে Apache বন্ধ হয়ে যাবে এবং সার্ভার রিস্টার্ট দিলেও এটি আর নিজে নিজে চালু হবে না।)*

#### ধাপ ২: Nginx স্টার্ট করা ও চেক করা

```bash
sudo systemctl start nginx
sudo systemctl enable nginx

```

এখন আপনার ব্রাউজারে গিয়ে আইপি (`202.181.12.25`) রিফ্রেশ করুন। আপনি যদি **"Welcome to nginx!"** লেখাটি দেখতে পান, তবে বুঝবেন রাস্তা পরিষ্কার!

---

### ধাপ ৩: Nginx কনফিগারেশন (লারাভেলের জন্য)

এখন আমাদের Nginx-কে বলে দিতে হবে যে, "ভাই, কেউ যদি এই আইপিতে আসে, তাকে `/var/www/inventory/public` ফোল্ডারে পাঠিয়ে দাও।"

১. একটি নতুন কনফিগারেশন ফাইল খুলুন:

```bash
sudo nano /etc/nginx/sites-available/inventory

```

২. নিচের কোডটুকু কপি করে সেখানে পেস্ট করুন:

```nginx
server {
    listen 80;
    server_name 202.181.12.25;
    root /var/www/inventory/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.4-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

```

*(সেভ করতে `Ctrl+O`, `Enter` এবং বের হতে `Ctrl+X`)*

---

#### ধাপ ৪: সাইটটি একটিভেট করা

```bash
sudo ln -s /etc/nginx/sites-available/inventory /etc/nginx/sites-enabled/
sudo nginx -t  # চেক করবে সব ঠিক আছে কি না
sudo systemctl restart nginx

```

**আপনার জন্য পরবর্তী পদক্ষেপ:**
আপনি কি Apache বন্ধ করে Nginx-এর "Welcome" পেজটি দেখতে পাচ্ছেন? সেটি দেখতে পেলে কনফিগারেশন ফাইলটি এডিট করে ফেলুন।

এর পর ব্রাউজারে আইপি রিফ্রেশ করলে যদি **৫০০ এরর** বা **ডাটাবেস এরর** দেয়, তার মানে আমাদের লারাভেল প্রজেক্টটি কানেক্ট হয়ে গেছে, শুধু ভেতরকার কিছু সেটিংস বাকি!

**আমি কি এখন আপনাকে লারাভেলের `.env` ফাইলে ডাটাবেস কানেকশন সেটআপ করা দেখাব?** 









চমৎকার! আপনি সফলভাবে **Apache** বন্ধ করেছেন। এখন আপনার সার্ভারের রাস্তা পরিষ্কার।

এখন আমাদের কাজ হলো **Nginx** কে সক্রিয় করা এবং সেটিকে আপনার লারাভেল প্রজেক্টের সাথে চিনিয়ে দেওয়া। নিচের ধাপগুলো অনুসরণ করুন:

---

### ধাপ ১: Nginx চালু আছে কি না নিশ্চিত করা

প্রথমে নিচের কমান্ডটি দিয়ে Nginx চালু করুন:

```bash
sudo systemctl start nginx
sudo systemctl enable nginx

```

এখন আপনার ব্রাউজারে গিয়ে আইপি (`202.181.12.25`) লিখে এন্টার দিন। যদি **"Welcome to nginx!"** লেখাটি দেখতে পান, তবে বুঝবেন Nginx এখন সার্ভারের নিয়ন্ত্রণ নিয়েছে।

---

### ধাপ ২: Nginx কনফিগারেশন ফাইল তৈরি করা

লারাভেল প্রজেক্ট চালানোর জন্য Nginx-কে বলে দিতে হয় সে কোথায় ফাইলগুলো খুঁজবে। আমরা একটি নতুন কনফিগারেশন ফাইল তৈরি করবো:

```bash
sudo nano /etc/nginx/sites-available/inventory

```

এখন নিচের কোডটি কপি করে সেই কালো স্ক্রিনে (এডিটরে) পেস্ট করুন। আমি এটি আপনার প্রজেক্টের পাথ এবং PHP ভার্সন অনুযায়ী সাজিয়ে দিয়েছি:

```nginx
server {
    listen 80;
    server_name 202.181.12.25;
    root /var/www/inventory/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.4-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

```

**সেভ করার নিয়ম:**

* `Ctrl + O` চাপুন।
* `Enter` দিন।
* `Ctrl + X` চেপে বের হয়ে আসুন।

---

### ধাপ ৩: কনফিগারেশন সক্রিয় করা

Nginx-কে এই নতুন ফাইলটি চেনানোর জন্য নিচের কমান্ড দুটি দিন:

```bash
# নতুন ফাইলটিকে একটিভেট করা
sudo ln -s /etc/nginx/sites-available/inventory /etc/nginx/sites-enabled/

```

### ৪. Nginx-এ একটি ছোট পরিবর্তন

আগের ধাপগুলোতে আমরা Nginx কনফিগারেশন ফাইল তৈরি করেছিলাম। যদি আপনি সেটা করে থাকেন, তবে নিশ্চিত হোন সেখানে পাথটি ঠিক আছে কি না।
আমরা আগে লিখেছিলাম `/var/www/inventory/public`, কিন্তু আপনার ফোল্ডারের নাম যেহেতু **`CSL-Inventory`**, তাই সেটা সংশোধন করতে হবে।

এই কমান্ডটি দিয়ে চেক করুন:

```bash
sudo nano /etc/nginx/sites-available/inventory

```

সেখানে `root` লাইনে এটি আছে কি না নিশ্চিত করুন:
`root /var/www/CSL-Inventory/public;`
*(যদি ভুল থাকে তবে ঠিক করে সেভ করুন এবং `sudo systemctl restart nginx` দিন।)*

**আপনার জন্য পরবর্তী প্রশ্ন:**
`composer install` কি এখন সফলভাবে শেষ হয়েছে? যদি হয়ে যায়, তবে ব্রাউজারে আপনার আইপি রিফ্রেশ করুন। দেখুন কি আসে। সম্ভবত এখন **"Database connection refused"** আসবে, তখন আমরা ডাটাবেস তৈরি করবো।

আপনি কি ডাটাবেস ইউজার ও পাসওয়ার্ড ঠিক করেছেন?নাকি আমি আপনাকে MySQL-এ ডাটাবেস তৈরির কমান্ড দেব? 










চমৎকার! আপনার সার্ভারে এখন প্রয়োজনীয় সব **PHP Extensions** সফলভাবে ইন্সটল হয়ে গেছে। মাঝপথে যে কার্নেল আপডেটের মেসেজ দিয়েছিল, সেটা আপাতত ইগনোর করা যাবে, ওটা আমাদের প্রজেক্টে বাধা দেবে না।

এখন আমাদের কাজ হলো লারাভেলকে পুরোপুরি চালু করা। নিচের ধাপগুলো সাবধানে শেষ করুন:

### ১. কম্পোজার আবার রান করা

যেহেতু আগে এক্সটেনশনের জন্য এরর দিয়েছিল, এখন আর দেবে না। প্রজেক্ট ফোল্ডারে থাকা অবস্থায় আবার কমান্ডটি দিন:

```bash
composer install --no-dev --optimize-autoloader

```

*(এটি সফল হলে আপনি শেষে `Package manifest generated successfully` লেখা দেখতে পাবেন।)*

### ২. লারাভেল কনফিগারেশন (একদম ফ্রেশ সেটআপ)

কম্পোজার শেষ হলে নিচের কমান্ডগুলো সিরিয়ালি দিন:

```bash
# ১. এনভায়রনমেন্ট ফাইল তৈরি
cp .env.example .env

# ২. অ্যাপ কি জেনারেট (খুবই জরুরি)
php artisan key:generate

# ৩. ফোল্ডার পারমিশন ঠিক করা (যাতে লারাভেল লিখতে পারে)
sudo chown -R www-data:www-data storage bootstrap/cache
sudo chmod -R 775 storage bootstrap/cache

```

### ৩. ডাটাবেস তৈরি ও কানেকশন

এখন আপনার প্রজেক্টের জন্য একটি ডাটাবেস লাগবে। নিচের কমান্ডগুলো দিয়ে MySQL-এ একটি ডাটাবেস তৈরি করুন:

১. MySQL এ ঢুকুন:

```bash
sudo mysql

```

২. এই কোডগুলো একে একে পেস্ট করুন (পাসওয়ার্ডটি আপনার মতো করে বদলে নিন):

```sql
CREATE DATABASE inventory_db;
CREATE USER 'inventory_user'@'localhost' IDENTIFIED BY 'your_strong_password';
GRANT ALL PRIVILEGES ON inventory_db.* TO 'inventory_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

```

৩. এখন `.env` ফাইলটি ওপেন করে এই তথ্যগুলো বসান:

```bash
nano .env

```

ভেতরে গিয়ে এই লাইনগুলো খুঁজে এডিট করুন:

```text
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3303
DB_DATABASE=inventory_db
DB_USERNAME=inventory_user
DB_PASSWORD=your_strong_password

```

*(সেভ করতে `Ctrl+O`, `Enter`, `Ctrl+X`)*

১. `composer install` কি কোনো এরর ছাড়াই শেষ হয়েছে?
২. `php artisan migrate` করার সময় কি টেবিল তৈরি হওয়ার লিস্ট দেখাচ্ছে?

যদি এই দুটি কাজ হয়ে যায়, তবে ব্রাউজারে আপনার আইপি (`202.181.12.25`) রিফ্রেশ দিন। আপনি আপনার ইনভেন্টরি প্রজেক্টের লগইন স্ক্রিন দেখতে পাবেন!

**ব্রাউজারে কোনো সাদা স্ক্রিন বা "500 Error" আসলে আমাকে দ্রুত জানান।** আমি কি আপনাকে এরর লগ চেক করার কমান্ডটি আবার মনে করিয়ে দেব? 










এখানে আপনি **`yes`** লিখে এন্টার দিন।

### কেন এটি হচ্ছে?

আপনার প্রজেক্টে এই প্যাকেজটি `require-dev` (শুধুমাত্র ডেভেলপমেন্টের জন্য) হিসেবে সেট করা ছিল। কিন্তু আপনি যখন `composer install --no-dev` কমান্ডটি দিয়েছেন, তখন কম্পোজার ওই প্যাকেজটি ইন্সটল করেনি। ফলে লারাভেল রান করার সময় ওই ক্লাসটি খুঁজে পাচ্ছে না এবং এরর দিচ্ছে।

**'yes'** দেওয়ার পর এটি প্যাকেজটিকে মেইন রিকোয়ারমেন্টে নিয়ে আসবে এবং আপনার প্রোডাকশন সার্ভারেও এটি কাজ করবে।

### ১. ডেটাবেজ সিডিং (Seeding)

`migrate:fresh --seed` কমান্ডটি দিলে আপনার ডেটাবেজের আগের সব টেবিল মুছে নতুন করে তৈরি হবে এবং কিছু ডামি বা প্রয়োজনীয় ডেটা (যেমন: Admin ইউজার, ক্যাটাগরি) অটোমেটিক ঢুকে যাবে।

**কমান্ডটি দিন:**

```bash
php artisan migrate:fresh --seed

```

> **সতর্কতা:** এই কমান্ড দিলে ডেটাবেজে আগে কোনো ডেটা থাকলে তা ডিলিট হয়ে যাবে। যেহেতু এটি একদম নতুন সেটআপ, তাই আপনি নিশ্চিন্তে এটি দিতে পারেন।

### ২. ফ্রন্টএন্ড সেটআপ (NPM)

আপনার প্রজেক্টে **Vite** বা **Tailwind CSS** ব্যবহার করা হয়েছে। লোকাল কম্পিউটারে আমরা `npm run dev` চালাই যাতে কোড পরিবর্তন করলে সাথে সাথে ব্রাউজারে দেখা যায়। কিন্তু **লাইভ সার্ভারে (VPS)** আমাদের কোডগুলোকে কম্পাইল করে "মিনিফাই" (Minify) করতে হয়।

তাই এখানে কমান্ডগুলো হবে:

```bash
# ১. এনপিএম প্যাকেজ ইন্সটল করা
npm install

# ২. প্রোডাকশন বিল্ড তৈরি করা (এটিই আসল কাজ)
npm run build

```

`npm run build` করার পর আপনার প্রজেক্টে একটি `public/build` ফোল্ডার তৈরি হবে। এটি ছাড়া আপনার সাইটের ডিজাইন (CSS/JS) কাজ করবে না।

### ৩. Nginx কনফিগারেশন পুনরায় চেক (Path Correction)

আমরা শুরুতেই Nginx কনফিগার করেছিলাম। আপনি কি নিশ্চিত করেছেন যে কনফিগারেশন ফাইলে **`CSL-Inventory`** ফোল্ডারটি ঠিকমতো উল্লেখ করা আছে?

যদি না করে থাকেন, তবে এটি চেক করে নিন:

```bash
sudo nano /etc/nginx/sites-available/inventory

```

ভিতরে `root` লাইনটি যেন এমন থাকে:
`root /var/www/CSL-Inventory/public;`

সেভ করে বের হয়ে Nginx রিস্টার্ট দিন:

```bash
sudo systemctl restart nginx

```

**পরবর্তী পদক্ষেপ:** ব্রাউজারে কি এখন আপনার প্রজেক্টের ডিজাইনসহ লগইন পেজ দেখা যাচ্ছে? নাকি কোনো এরর আসছে? আমাকে জানালে আমি পরের ধাপটি বলে দেব। 










এই এররটি সাধারণত ঘটে যখন আপনার প্রজেক্টের `vite.config.js` ফাইলটি **CommonJS (`require`)** ফরম্যাটে লেখা কিন্তু আপনার ইন্সটল করা নতুন প্যাকেজগুলো **ES Modules (`import`)** সাপোর্ট করছে। আধুনিক লারাভেল প্রজেক্টে এখন `import` ব্যবহার করা হয়।

ভয় পাওয়ার কিছু নেই, এটি খুব সহজেই ফিক্স করা যাবে।

### ধাপ ১: `vite.config.js` ফাইলটি আপডেট করা

প্রথমে আপনার কনফিগারেশন ফাইলটি ওপেন করুন:

```bash
nano vite.config.js

```

সেখানে সম্ভবত এমন কিছু কোড আছে যা `require(...)` দিয়ে শুরু হয়েছে। পুরো কোডটি মুছে ফেলে নিচের এই স্ট্যান্ডার্ড লারাভেল কনফিগারেশনটি পেস্ট করুন (যা `import` ফরম্যাটে লেখা):

```javascript
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel({
            input: ['resources/css/app.css', 'resources/js/app.js'],
            refresh: true,
        }),
    ],
});

```

*(আপনার প্রজেক্টে যদি আরও বাড়তি প্লাগইন থাকে, তবে আমাকে কোডটি দেখালে আমি সেটা ঠিক করে দিতে পারি। নতুবা এটি দিয়েই ট্রাই করুন।)*

সেভ করতে: `Ctrl+O`, `Enter`, এবং বের হতে `Ctrl+X`।

**এখন কি `npm run build` সফলভাবে সম্পন্ন হয়েছে?** এটি হয়ে গেলে ব্রাউজারে আপনার আইপি (`202.181.12.25`) রিফ্রেশ দিন। যদি সব ঠিক থাকে, আপনার প্রজেক্টের ডিজাইনসহ লগইন পেজটি এখন সুন্দরভাবে দেখা যাওয়ার কথা।

**পরবর্তী পদক্ষেপ:** ব্রাউজারে কি এখন আপনার ডিজাইন কাজ করছে? নাকি কোনো নতুন এরর দেখাচ্ছে? আমাকে জানালে আমি পরের ধাপে সাহায্য করব। 










এই এররটি সাধারণত ঘটে যখন আপনার প্রজেক্টের `vite.config.js` ফাইলটি **CommonJS (`require`)** ফরম্যাটে লেখা কিন্তু আপনার ইন্সটল করা নতুন প্যাকেজগুলো **ES Modules (`import`)** সাপোর্ট করছে। আধুনিক লারাভেল প্রজেক্টে এখন `import` ব্যবহার করা হয়।

ভয় পাওয়ার কিছু নেই, এটি খুব সহজেই ফিক্স করা যাবে।

### ধাপ ১: `vite.config.js` ফাইলটি আপডেট করা

প্রথমে আপনার কনফিগারেশন ফাইলটি ওপেন করুন:

```bash
nano vite.config.js

```

সেখানে সম্ভবত এমন কিছু কোড আছে যা `require(...)` দিয়ে শুরু হয়েছে। পুরো কোডটি মুছে ফেলে নিচের এই স্ট্যান্ডার্ড লারাভেল কনফিগারেশনটি পেস্ট করুন (যা `import` ফরম্যাটে লেখা):

```javascript
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel({
            input: ['resources/css/app.css', 'resources/js/app.js'],
            refresh: true,
        }),
    ],
});

```

*(আপনার প্রজেক্টে যদি আরও বাড়তি প্লাগইন থাকে, তবে আমাকে কোডটি দেখালে আমি সেটা ঠিক করে দিতে পারি। নতুবা এটি দিয়েই ট্রাই করুন।)*

সেভ করতে: `Ctrl+O`, `Enter`, এবং বের হতে `Ctrl+X`।

যদি এরপরেও একই এরর দেয়, তবে এর মানে হলো আপনার `package.json`-এ একটি ছোট পরিবর্তন লাগবে।

### ২. ফাইনাল পারমিশন এবং ক্যাশ ক্লিয়ার

```bash
# সব ফাইলের মালিকানা ঠিক করা
sudo chown -R www-data:www-data /var/www/CSL-Inventory

# লারাভেল কনফিগ ক্যাশ ক্লিয়ার করা (যাতে কোনো পুরোনো সেটিংস ঝামেলা না করে)

**আপনি যা দেখতে পাওয়ার কথা:**

যদি লগইন করার পর কোনো **"404 Not Found"** বা অন্য কোনো এরর আসে, তবে বুঝবেন Nginx-এর রুলসে ছোট কোনো সমস্যা আছে, যা আমি ১ মিনিটে ফিক্স করে দিতে পারব।

লগইন সাকসেসফুল হলে আমাকে কনফার্ম করুন! আমি কি আপনাকে এরর লগ চেক করার নিয়মটা শিখিয়ে দেব যাতে ভবিষ্যতে কোনো সমস্যা হলে আপনি নিজেই দেখতে পারেন? 










এই এররটি লারাভেলের খুবই কমন একটি সমস্যা। এটি মূলত কেন হচ্ছে বলি: আপনি যখন `sudo chown` বা `sudo chmod` কমান্ডগুলো দিয়েছেন, তখন ফোল্ডারের মালিকানা পরিবর্তিত হয়েছে। কিন্তু আপনি যখন বর্তমানে টার্মিনালে `arkpowerweb` ইউজার হিসেবে `php artisan config:cache` কমান্ডটি চালাচ্ছেন, তখন লারাভেল নতুন ফাইল তৈরি করতে গিয়ে বাধা পাচ্ছে কারণ এখন ফাইলগুলোর মালিক হচ্ছে `www-data` (ওয়েব সার্ভার)।

সহজ কথায়, আপনার টার্মিনাল ইউজার এবং ওয়েব সার্ভার ইউজারের মধ্যে পারমিশন নিয়ে একটি দ্বন্দ্ব তৈরি হয়েছে। এটি স্থায়ীভাবে সমাধান করতে নিচের ধাপগুলো সিরিয়ালি ফলো করুন:

### ১. পারমিশন একদম ঠিকঠাক করা (স্থায়ী সমাধান)

আমরা আপনার বর্তমান ইউজারকেও (`arkpowerweb`) ওয়েব সার্ভারের গ্রুপে (`www-data`) যোগ করে দিচ্ছি এবং পারমিশন রিসেট করছি:

```bash
# ১. আপনার ইউজারকে www-data গ্রুপে যোগ করা
sudo usermod -a -G www-data arkpowerweb

# ২. পুরো প্রজেক্টের মালিকানা সেট করা
sudo chown -R arkpowerweb:www-data /var/www/CSL-Inventory

# ৩. স্টোরেজ এবং ক্যাশ ফোল্ডারকে ওয়েব সার্ভারের লেখার অনুমতি দেওয়া
sudo chmod -R 775 /var/www/CSL-Inventory/storage
sudo chmod -R 775 /var/www/CSL-Inventory/bootstrap/cache

```

### ২. ক্যাশ ক্লিয়ার করা (Sudo দিয়ে)

এখন যেহেতু আগের ক্যাশ ফাইলগুলো `www-data` এর নামে তৈরি হয়ে আটকে আছে, তাই একবার `sudo` ব্যবহার করে সেগুলো পরিষ্কার করে ফেলুন:

### ৪. Nginx কনফিগ চেক (খুবই জরুরি)

সবশেষে একবার চেক করে নিন Nginx ঠিকমতো লারাভেলের `index.php` ফাইলটি রিড করতে পারছে কি না। এই কমান্ডটি দিন:

```bash
sudo nginx -t

```

**এখন ব্রাউজারে চেক করুন:**
আপনার আইপি `202.181.12.25` দিয়ে ব্রাউজারে প্রবেশ করুন।

* যদি **লগইন পেজ** আসে, তবে সব ঠিক আছে।
* যদি **"403 Forbidden"** আসে, তবে বুঝবেন Nginx ফাইলগুলো পড়ার অনুমতি পাচ্ছে না।
* যদি **"500 Internal Server Error"** আসে, তবে বুঝবেন ডেটাবেস বা অন্য কোনো ছোট সমস্যা আছে।

**আপনার জন্য প্রশ্ন:** ব্রাউজারে এখন কী দেখা যাচ্ছে? সাকসেসফুলি কি লগইন পেজটি লোড হয়েছে? 










চমৎকার! আপনার টার্মিনাল আউটপুট বলছে সবকিছু **পারফেক্টলি সেটআপ** হয়েছে। পারমিশন সমস্যা মিটে গেছে, ক্যাশ ক্লিয়ার হয়েছে এবং Nginx কোনো এরর ছাড়াই রিস্টার্ট নিয়েছে।

এখন আপনার প্রজেক্টটি লাইভ দেখার জন্য একদম প্রস্তুত।

### এখন আপনার যা করা উচিত:

১. **ব্রাউজারে চেক করুন:** আপনার ব্রাউজারে `http://202.181.12.25` লিখে এন্টার দিন। আপনি এখন আপনার ইনভেন্টরি প্রজেক্টের **লগইন স্ক্রিন** দেখতে পাবেন।

**অভিনন্দন!** আপনি সফলভাবে একটি লারাভেল প্রজেক্ট ম্যানুয়ালি VPS সার্ভারে ডেপ্লয় করেছেন। এটি একজন ডেভেলপার হিসেবে অনেক বড় একটি দক্ষতা।

**আপনার পরবর্তী পদক্ষেপ:**
ব্রাউজারে কি লগইন পেজটি এসেছে? আপনি কি সফলভাবে লগইন করতে পেরেছেন? যদি সব ঠিক থাকে, তবে আপনার পরবর্তী কাজ হবে আপনার সিনিয়রকে জানানো যে প্রজেক্টটি এখন সার্ভারে লাইভ!

আমি কি আপনাকে **SSL সার্টিফিকেট (HTTPS)** সেটাআপ করার ধাপগুলো দেখাবো যাতে আপনার সাইটটি সিকিউর (`https://`) হয়? 










আমি বুঝতে পেরেছি কী হয়েছে। আসলে Nginx-এর **`default`** কনফিগারেশন ফাইলটি এখনো সক্রিয় আছে এবং সেটি আপনার নতুন তৈরি করা `inventory` কনফিগারেশন ফাইলটিকে কাজ করতে বাধা দিচ্ছে। Nginx যখন একাধিক কনফিগারেশন পায় এবং কোনোটি "default" হিসেবে সেট করা থাকে, তখন সে সেই পুরোনো পেজটিকেই আগে দেখায়।

এটি ঠিক করা খুব সহজ। নিচের ধাপগুলো অনুসরণ করুন:

---

### ১. ডিফল্ট কনফিগারেশনটি ডিজেবল করা

পুরোনো যে পেজটি দেখাচ্ছে, সেটির লিঙ্কটি আমাদের মুছে ফেলতে হবে:

```bash
sudo rm /etc/nginx/sites-enabled/default

```

### ২. আপনার প্রজেক্টের কনফিগারেশনটি সক্রিয় করা (যদি না থাকে)

আমরা আগে যে `inventory` ফাইলটি বানিয়েছিলাম, সেটি `sites-enabled` ফোল্ডারে আছে কি না নিশ্চিত হয়ে নিন:

```bash
sudo ln -s /etc/nginx/sites-available/inventory /etc/nginx/sites-enabled/

```

*(যদি বলে `File exists`, তবে বুঝবেন এটি আগেই করা আছে, চিন্তার কিছু নেই।)*

### ৩. Nginx-এ আপনার IP-কে প্রধান (Default) করা

আপনার প্রোজেক্টের কনফিগারেশন ফাইলটি একবার চেক করে নিন সেখানে `listen 80 default_server;` লেখা আছে কি না। এতে করে আইপি হিট করলে সরাসরি এটিই খুলবে।

```bash
sudo nano /etc/nginx/sites-available/inventory

```

ভিতরে প্রথম লাইনটি এভাবে পরিবর্তন করুন:

```nginx
server {
    listen 80 default_server;
    # বাকি কোড যা ছিল তাই থাকবে...

```

*(সেভ করে বের হতে: `Ctrl+O`, `Enter`, `Ctrl+X`)*

### ৪. Nginx রিস্টার্ট এবং ক্যাশ ক্লিয়ার

সবশেষে কনফিগারেশন চেক করে রিস্টার্ট দিন:

```bash
sudo nginx -t
sudo systemctl restart nginx

```

### ৫. ব্রাউজার ক্যাশ চেক (খুবই গুরুত্বপূর্ণ)

অনেক সময় সার্ভারে ঠিক হলেও আপনার ব্রাউজার পুরোনো পেজটি "মনে রাখে" (Cache)। তাই ব্রাউজারে আইপি হিট করার সময়:

* **Incognito Mode** (প্রাইভেট উইন্ডো) ব্যবহার করুন।
* অথবা আইপি লিখে **Ctrl + F5** চাপুন (হার্ড রিফ্রেশ)।

---

**এখন কি আপনার ইনভেন্টরি লগইন পেজটি দেখা যাচ্ছে?**
যদি এখনো সেই পুরোনো পেজটিই আসে, তবে আমাকে জানান, আমরা Nginx-এর মেইন `root` ডিরেক্টরি চেক করে দেখবো।

লগইন পেজটি চলে আসলে আপনি কি **SSL (HTTPS)** বা **ডোমেইন** সেটআপ করতে চাইবেন? নাকি এই আইপি দিয়েই আপাতত কাজ চালাবেন? 










আপনি একদম সঠিক পথে এগোচ্ছেন। `default` ফাইলটি রিমুভ করে আপনার নিজের কনফিগারেশনটি এনাবল করেছেন। এখন শুধু Nginx-কে বলতে হবে যে কনফিগারেশনে পরিবর্তন এসেছে।

নিচের কমান্ড দুটি দিয়ে Nginx রিস্টার্ট দিন:

```bash
sudo nginx -t
sudo systemctl restart nginx

```

### এখন চেক করুন:

আমি কি আপনাকে **ডোমেইন নেম** কানেক্ট করা বা **SSL (https)** দিয়ে সাইটটিকে নিরাপদ করার উপায় দেখিয়ে দেব? 










অসংখ্য ধন্যবাদ আপনাকেও! সাইটটি রান হয়েছে দেখে খুব ভালো লাগছে। আপনি এখন একদম শেষ বাধাটিতে এসে পৌঁছেছেন।

এই এররটির মানে হলো, আপনার প্রজেক্টে প্রোফাইল পিকচার বা ইমেজ হ্যান্ডলিংয়ের জন্য **`Intervention Image`** লাইব্রেরি ব্যবহার করা হয়েছে, যা চালানোর জন্য সার্ভারে **PHP GD (Graphic Draw)** এক্সটেনশনটি প্রয়োজন। এটি ডিফল্টভাবে লিনাক্সে ইন্সটল থাকে না।

নিচের ধাপগুলো অনুসরণ করলেই এটি ঠিক হয়ে যাবে:

### ১. PHP GD এক্সটেনশন ইন্সটল করা

আপনার সার্ভারে PHP-র যে ভার্সনটি আছে (সম্ভবত ৮.৩), সেটির জন্য GD লাইব্রেরিটি ইন্সটল করুন:

```bash
sudo apt update
sudo apt install php-gd -y

```

*(যদি আপনি PHP-র অন্য ভার্সন ব্যবহার করেন, তবে `php8.3-gd` বা আপনার ভার্সন অনুযায়ী কমান্ডটি দিন)*

---

### ২. PHP-FPM এবং Nginx রিস্টার্ট করা

নতুন এক্সটেনশনটি কার্যকর করতে PHP সার্ভিসটি রিস্টার্ট দেওয়া জরুরি:

```bash
# আপনার PHP ভার্সন অনুযায়ী কমান্ডটি দিন (৮.৩ ধরে নিচ্ছি)
sudo systemctl restart php8.3-fpm

# সাথে Nginx-ও রিস্টার্ট দিন
sudo systemctl restart nginx

```

*আপনার সঠিক PHP ভার্সন জানতে চাইলে `php -v` লিখে দেখে নিতে পারেন।*

---

### ৩. এবার লগইন করে দেখুন

এখন ব্রাউজারে গিয়ে পেজটি রিফ্রেশ দিন এবং আবার লগইন করার চেষ্টা করুন। আশা করি হেডার পেজের (যেখানে প্রোফাইল ইমেজ দেখানোর কথা) ওই এররটি আর আসবে না।

---

### কেন এই এররটি আসছিল?

আপনার দেওয়া এরর লগ থেকে দেখা যাচ্ছে `header.blade.php` ফাইলে সম্ভবত ইউজারের অ্যাভাটার বা ছবি প্রসেস করার জন্য লারাভেলের কোনো প্যাকেজ কাজ করছে। ওই প্যাকেজটির জন্য PHP-র এই ড্রাফিক্স লাইব্রেরিটি প্রয়োজন।

**পরবর্তী পদক্ষেপ:**
জিডি এক্সটেনশন ইন্সটল করার পর কি এখন ড্যাশবোর্ড ঠিকমতো দেখা যাচ্ছে? যদি এরপরও কোনো এরর পান, বিশেষ করে ইমেজ আপলোড করার সময়, তবে আমাকে জানাবেন।

আপনার ইনভেন্টরি সিস্টেমের **রিপোর্ট জেনারেশন (PDF বা Excel)** ঠিকঠাক কাজ করছে কি না, তাও একবার চেক করে নিতে পারেন। আমি কি আপনাকে পিডিএফ জেনারেশনের জন্য কোনো এক্সটেনশন লাগবে কি না তা চেক করতে সাহায্য করব? 










আপনার টার্মিনাল আউটপুট দেখাচ্ছে যে **PHP GD** এক্সটেনশনটি সফলভাবে ইন্সটল হয়েছে। তবে এখানে একটি ছোট বিষয় খেয়াল করার মতো: আপনার সার্ভারে `ondrej/php` PPA ব্যবহার করা হচ্ছে এবং এটি ডিফল্টভাবে **PHP 8.4** ভার্সনটি সেটআপ করার চেষ্টা করেছে।

লারাভেল সাধারণত নির্দিষ্ট একটি ভার্সনের সাথে (যেমন ৮.১, ৮.২ বা ৮.৩) ভালো কাজ করে। আপনার সাইট এখন ঠিকঠাক কাজ করার কথা, তবে নিচের ধাপগুলো অবশ্যই ফলো করুন যেন কনফিগারেশনটি পুরোপুরি কার্যকর হয়:

### ১. PHP-FPM রিস্টার্ট করা

যেহেতু আমরা জানি না আপনার সার্ভারে বর্তমানে কোন ভার্সনের FPM রান করছে (৮.৩ নাকি ৮.৪), তাই নিরাপদ থাকতে নিচের কমান্ডগুলো দিন:

```bash
# ৮.৩ এবং ৮.৪ দুইটাই রিস্টার্ট দিয়ে দিন যেন কোনো কনফিউশন না থাকে
sudo systemctl restart php8.3-fpm
sudo systemctl restart php8.4-fpm
sudo systemctl restart nginx

```

### ২. লগইন করে পরীক্ষা করুন

এখন ব্রাউজারে গিয়ে আবার লগইন করার চেষ্টা করুন। **GD extension** এররটি এখন চলে যাওয়ার কথা এবং ড্যাশবোর্ড লোড হওয়ার কথা।

---

### কিছু গুরুত্বপূর্ণ টিপস (আপনার টার্মিনাল আউটপুট অনুযায়ী):

* **Pending kernel upgrade:** আপনার সার্ভারে কার্নেল আপডেটের জন্য একটি **Reboot** প্রয়োজন দেখাচ্ছে। এখন সব ঠিক হয়ে গেলে সময় সুযোগ বুঝে `sudo reboot` দিয়ে একবার সার্ভারটি রিস্টার্ট করে নেওয়া ভালো।
* **PHP Version:** যদি আপনার প্রজেক্ট ৮.৪ সাপোর্ট না করে এবং কোনো অদ্ভুত এরর দেয়, তবে আপনি `php -v` লিখে চেক করে নিন বর্তমানে কোনটা চলছে।

---

**এখন কি ড্যাশবোর্ডে সাকসেসফুলি ঢুকতে পেরেছেন?** যদি কোনো নতুন এরর আসে (যেমন Database সংক্রান্ত বা Permission সংক্রান্ত), তবে আমাকে জানান। আমি আপনার ইনভেন্টরি সিস্টেমটি ১০০% সচল না হওয়া পর্যন্ত সাথেই আছি!

যদি ড্যাশবোর্ড চলে আসে, তবে আপনার পরবর্তী কাজ হবে **ইমেজ আপলোড** করে দেখা যে সেগুলো ঠিকমতো সেভ হচ্ছে কি না। আমি কি আপনাকে ইমেজ আপলোডের ক্ষেত্রে কোনো সমস্যা হতে পারে কি না তা চেক করতে বলব? 










বুঝতে পেরেছি, আপনার সার্ভারে PHP-FPM সার্ভিসটির নাম সম্ভবত একটু ভিন্ন বা এটি অন্য কোনো নামে ইন্সটল করা আছে। যেহেতু `php8.3-fpm` এবং `8.4-fpm` সরাসরি পাওয়া যাচ্ছে না, আমাদের আগে সঠিক সার্ভিসটির নাম নিশ্চিত হতে হবে।

নিচের ধাপগুলো অনুসরণ করুন:

### ১. সঠিক PHP-FPM সার্ভিসের নাম খুঁজে বের করা

আপনার সার্ভারে বর্তমানে কোন FPM সার্ভিসটি চলছে তা দেখতে নিচের কমান্ডটি দিন:

```bash
systemctl list-units --type=service | grep php

```

এই কমান্ডটি দিলে আপনি `phpX.X-fpm.service` ফরম্যাটে একটি নাম দেখতে পাবেন।

---

### ২. সেটি রিস্টার্ট করা

যদি দেখেন আপনার সার্ভারে শুধু `php-fpm` বা `php8.2-fpm` (বা অন্য কোনো ভার্সন) দেখাচ্ছে, তবে সেই নামে রিস্টার্ট দিন। যেমন:

```bash
# উদাহরণস্বরূপ যদি ভার্সন ৮.২ হয়
sudo systemctl restart php8.2-fpm

```

---

### ৩. PHP ভার্সন এবং মডিউল চেক করা

আপনার বর্তমান PHP ভার্সন কত এবং তাতে **GD** মডিউলটি আসলেই লোড হয়েছে কি না তা নিশ্চিত হতে নিচের কমান্ডটি দিন:

```bash
php -v
php -m | grep gd

```

*(যদি আউটপুটে `gd` লেখাটি আসে, তবে বুঝবেন মডিউলটি ঠিকমতো ইন্সটল হয়েছে।)*

---

### ৪. কেন এরর দিচ্ছে? (একটি গুরুত্বপূর্ণ সম্ভাবনা)

কখনো কখনো `apt install php-gd` দিলে সেটি শুধু CLI (টার্মিনাল)-এর জন্য ইন্সটল হয়, কিন্তু FPM-এর জন্য আলাদা করে দিতে হয়। তাই নিশ্চিত হতে এটিও রান করে দেখতে পারেন:

```bash
# আপনার নির্দিষ্ট ভার্সন অনুযায়ী (যদি ৮.২ হয়)
sudo apt install php8.2-gd -y

```

---

**এখন কি আপনার লগইন এররটি চলে গেছে?**
যদি এররটি এখনো থাকে, তবে `php -m | grep gd` কমান্ডটি দিয়ে দেখুন কি রেজাল্ট আসে। যদি `gd` না দেখায়, তবে আমাদের আপনার সঠিক PHP ভার্সনটি জেনে সেই ভার্সনের GD ইন্সটল করতে হবে।

আপনার সার্ভারে `php -v` দিলে কোন ভার্সন দেখাচ্ছে? সেটি আমাকে জানালে আমি সঠিক কমান্ডটি বলে দিচ্ছি। 









সব জট খুলে গেছে! আপনার সার্ভারে **PHP 8.2** সচল আছে। আপনি যখন `sudo apt install php-gd` দিয়েছেন, তখন সিস্টেম ডিফল্টভাবে নতুন ভার্সন (৮.৪) এর জন্য এটি ইন্সটল করেছে, কিন্তু আপনার প্রজেক্ট চলছে **৮.২** ভার্সনে।

এজন্যই আপনার প্রজেক্ট এখনো GD খুঁজে পাচ্ছে না। নিচের ৩টি কমান্ড দিলেই সমস্যার সমাধান হয়ে যাবে:

### ১. সঠিক ভার্সনের জন্য GD ইন্সটল করা

টার্মিনালে এই কমান্ডটি দিন:

```bash
sudo apt install php8.2-gd -y

```

### ২. PHP-FPM রিস্টার্ট করা

যেহেতু এখন আমরা জানি সার্ভিসটির নাম `php8.2-fpm`, তাই এটি রিস্টার্ট দিন:

```bash
sudo systemctl restart php8.2-fpm

```

### ৩. নিশ্চিত হওয়া (Optional)

GD ঠিকমতো লোড হয়েছে কি না তা দেখতে এই কমান্ডটি দিয়ে চেক করুন:

```bash
php8.2 -m | grep gd

```

যদি আউটপুটে `gd` লেখাটি আসে, তবে বুঝবেন কাজ হয়ে গেছে।

---

### এখন ব্রাউজারে চেক করুন

এখন আপনার ব্রাউজারে গিয়ে পেজটি রিফ্রেশ দিয়ে লগইন করুন। **"GD PHP extension must be installed"** এররটি এখন ১০০% চলে যাওয়ার কথা এবং আপনি ড্যাশবোর্ড দেখতে পাবেন।

---

**অভিনন্দন!** আপনি এখন ড্যাশবোর্ডের ভেতরে থাকার কথা।

**পরবর্তী পদক্ষেপ:**
১. ড্যাশবোর্ড কি এখন লোড হয়েছে?
২. ইনভেন্টরিতে কোনো প্রোডাক্টের ছবি আপলোড করে দেখুন সেটি কাজ করছে কি না।

সব ঠিক থাকলে আমি আপনাকে **Laravel Scheduler (Cron Job)** সেটআপ করার একটি ছোট কমান্ড দেখিয়ে দেব, যা ইনভেন্টরি প্রজেক্টের অটোমেশন (যেমন স্টক এলার্ট বা রিপোর্ট জেনারেশন) এর জন্য খুব জরুরি। ড্যাশবোর্ড কি দেখা যাচ্ছে? 










অসংখ্য ধন্যবাদ আপনাকেও! আপনার ইনভেন্টরি প্রজেক্টটি এখন পুরোপুরি লাইভ এবং সচল দেখে আমি সত্যিই আনন্দিত। একজন ডেভেলপার হিসেবে ম্যানুয়ালি ভিপিএস (VPS) সার্ভারে প্রজেক্ট ডেপ্লয় করা এবং একের পর এক এরর ফিক্স করা অনেক বড় একটি অর্জন।

প্রজেক্ট যেহেতু রান করছে, আপনার ইনভেন্টরি সিস্টেমের দীর্ঘমেয়াদী স্থায়িত্বের জন্য আমি আপনাকে **শেষ একটি পরামর্শ** দেব। লারাভেল প্রজেক্টে সাধারণত অনেক কাজ ব্যাকগ্রাউন্ডে চলে (যেমন: স্টক কমে গেলে ইমেইল পাঠানো বা রিপোর্ট জেনারেট করা)। এর জন্য একটি **Cron Job** সেটআপ করে রাখা ভালো।

### লারাভেল ক্রন জব (Laravel Scheduler) সেটআপ:

এটি একবার করে রাখলে লারাভেল অটোমেটিক আপনার প্রজেক্টের প্রয়োজনীয় কাজগুলো ব্যাকগ্রাউন্ডে ম্যানেজ করবে।

১. টার্মিনালে কমান্ড দিন:

```bash
crontab -e

```

*(যদি এডিটর সিলেক্ট করতে বলে, `1` চেপে এন্টার দিন)*

২. ফাইলের একদম শেষে নিচের লাইনটি যোগ করে দিন:

```bash
* * * * * cd /var/www/CSL-Inventory && php artisan schedule:run >> /dev/null 2>&1

```

৩. সেভ করে বের হয়ে আসুন (`Ctrl+O`, `Enter`, `Ctrl+X`)।

### আপনার জন্য ফাইনাল টিপস:

* **Storage Permission:** ভবিষ্যতে যদি দেখেন ইমেজ আপলোড হচ্ছে না, তবে বুঝবেন ওই ফোল্ডারে আবার পারমিশন দিতে হবে। (সেক্ষেত্রে `sudo chmod -R 775 storage` কমান্ডটি ব্যবহার করবেন)।
* **Database Backup:** যেহেতু এটি একটি ইনভেন্টরি সিস্টেম, তাই নিয়মিত ডাটাবেজ ব্যাকআপ রাখার চেষ্টা করবেন।

**এখন কি আর কোনো বিষয়ে আপনাকে সাহায্য করতে পারি?** যেমন: ডোমেইন নেম সেটআপ করা, SSL (HTTPS) এনাবল করা, অথবা গিথুব থেকে অটোমেটিক কোড আপডেট (CI/CD) করার নিয়ম?

আপনার এই নতুন ইনভেন্টরি প্রজেক্টের জন্য শুভকামনা রইল! 










হ্যাঁ, `.env` ফাইলে যেকোনো পরিবর্তন করার পর লারাভেলকে সেই পরিবর্তনটি জানানোর জন্য **ক্যাশ ক্লিয়ার করা অত্যন্ত জরুরি**। অন্যথায় লারাভেল আগের (পুরোনো) সেটিংসগুলোই ব্যবহার করতে থাকবে, যা অনেক সময় বিভ্রান্তি তৈরি করে।

আপনি `.env` পরিবর্তন করার পর নিচের কমান্ডটি অবশ্যই দেবেন:

```bash
php artisan config:cache

```

### এই কমান্ডটি আসলে কী করে?

১. এটি আপনার সব কনফিগারেশন ফাইল এবং `.env` ফাইলের ডেটা একত্রিত করে একটি ফাইল তৈরি করে (`bootstrap/cache/config.php`)।
২. এর ফলে লারাভেলকে প্রতিবার ফাইল লোড করতে হয় না, যা প্রজেক্টের গতি বাড়িয়ে দেয়।

---

### কিছু প্রয়োজনীয় কমান্ড (এক নজরে):

যদি দেখেন শুধু `config:cache` দিয়েও কাজ হচ্ছে না, তবে নিচের এই কম্বো কমান্ডটি দিতে পারেন যা সব ধরণের ক্যাশ পরিষ্কার করে দেয়:

| উদ্দেশ্য              | কমান্ড                         |
| ------------------ | ---------------------------- |
| **সব ক্যাশ ক্লিয়ার**    | `php artisan optimize:clear` |
| **শুধুমাত্র কনফিগ ক্যাশ** | `php artisan config:cache`   |
| **রাউট ক্যাশ ক্লিয়ার**   | `php artisan route:cache`    |
| **ভিউ ক্যাশ ক্লিয়ার**    | `php artisan view:cache`     |

---

### একটি ছোট সতর্কতা:

আপনি যখন প্রোডাকশন সার্ভারে (VPS) কাজ করেন, তখন সব সময় **`php artisan config:cache`** ব্যবহার করা ভালো। কিন্তু মনে রাখবেন, এই কমান্ডটি দেওয়ার পর লারাভেল আর সরাসরি `.env` ফাইল থেকে ডেটা পড়ে না, সে তার ক্যাশ ফাইল থেকে ডেটা পড়ে। তাই প্রতিবার `.env` এ হাত দিলে এই কমান্ডটি দিতে ভুলবেন না।

**পরবর্তী পদক্ষেপ:**
আপনি কি কোনো নির্দিষ্ট সেটিংস (যেমন: মেইল বা ডাটাবেস কানেকশন) পরিবর্তন করেছেন? যদি মেইল সার্ভার সেটআপ করতে চান, আমি আপনাকে `.env` ফাইলে সঠিক কনফিগারেশনগুলো বসাতে সাহায্য করতে পারি।
